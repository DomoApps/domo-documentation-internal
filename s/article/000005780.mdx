title: "Process Instance Deletion Requests"
---

### Intro

From time to time, Support may receive a request to delete an instance for a customer. There are three typical types of requests. Learn how to process each type below.

---

### Delete a Freemium Instance

Support can only delete true Freemium instances. To check, you may need to log in to Admin Prod, enter the instance, and check the instance tier. If it's a Freemium tier, submit a Hivemind post for the L2 team to delete the instance.

<Note>**Note:** A request to delete a Freemium instance must be made IN WRITING from an Admin in the instance.</Note>

#### For L2s

If you see a request to delete a Freemium instance, follow these steps:

1. Log in to Yoda.
2. Select **Domo Instances** in the left nav bar.
3. Enter the instance domain in the search bar.
4. Select the domain that displays in the results.
5. If this is an actual Freemium instance, a **Delete** button displays in the top right corner.

### Delete a Subscriber Instance

<Warning>**Important:** Support should *not* do this for the customer.</Warning>

The customer should do this themselves within the Instance Management section of their Admin panel. If there are issues preventing the customer from deleting the instance, please post a question on the Hivemind board so an L2 can review possible causes.

### Delete a Regular Paid Instance

<Warning>**Important:** Support cannot delete these instances and we do not process these requests.</Warning>

If a customer wants to delete a paid instance, they need to engage with their account team. The account team must contact their AR specialist to process the request. The account team should *not* reach out to the instances team. They cannot delete paid instances either.
