### Intro

After an L1 Advisor has completed the basic troubleshooting of a connector-related issue, if the customer needs more information or has questions, your next step should be to engage with the Connector L3 team.

---

### When to Involve the Connector L3 Team

The Connector L3 team can help in the following situations:

- API call requests
- Connector feasibility checks
- Determine if a field is available in a report
- API documentation review
- JSON No Code connector review
- Basic Workbench error reviews
- File Upload and DataSet via Email cases
  - 
<Note>**Note:** The .xlsx, .csv, .txt, or .eml file will be required to investigate
- Writeback connector errors
  - **Note:** They will not be able to run the DataSet in troubleshooting due to the potential of overwriting the customer's production data.</Note>

L1 Advisors should not send the following issues to the L3 team:

- Writeback connector issues where the DataSet must have a DataSet update run
- On-premise configurations (Database connectors)
- Federated DataSets or Cloud Amplifier
- Enable Instagram Business Connector requests; these follow a different process. Begin by reading about the [Instagram Business Connector](/s/article/360042929754) in the Knowledge Base.

### Create JIRA Story Ticket

When engaging with the L3 team, the L1 Advisor should retain ownership over the Salesforce case. Create a **Story** JIRA ticket for the L3 team.

#### Fill out the following JIRA ticket fields:

- Project: Domo (DOMO)
- Issue Type: Story
- Summary: <*connector name*> | <*request type*>
- Priority:
  - P3: Default priority (used for most cases)
  - P2: Escalated or High cases; used to prioritize work on these cases
- Components: Connectors
- Assignee: Tejmani Singh
- Squad: Support
- Description (see [templates](#description_templates) below)
- Attachment: Include files for File Upload and DataSet via Email cases or Workbench-related questions
- Environment: Prod number of the instance
- Connector ID: Provide the connector ID based on the customer DataSet; this should have been obtained when cloning the connector for initial testing
