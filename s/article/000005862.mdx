### Intro

Understanding how the Pro-code Editor works is essential for effectively troubleshooting customer issues. The Pro-code Editor is the successor to Domo Bricks, sharing similar functionality but offering greater flexibility. Unlike bricks, which each included three base files, the Pro-code Editor lets you add unlimited files organized in a folder structure like Visual Studio.

### Key Features of the Pro-code Editor

##### Versioning

Versioning enables tracking and managing different versions of resources like DataSets, visualizations, or apps. When enabled, Domo keeps a history of changes, allowing users to access and revert to prior versions if needed. Benefits include:

- Change tracking
- Rollback capability
- Collaboration and review
- Testing and experimentation
- Archiving and documentation

##### Graphical User Interface (GUI)

The built-in GUI assists with JavaScript development and reduces the need for extensive coding expertise.

##### Manifest File (manifest.json)

This file allows Domo to better interact with the app, managing metadata and configuration.

### Troubleshooting Steps

Most issues unrelated to the interface or folder system usually stem from code errors or problematic external calls. Follow these steps to isolate and resolve problems efficiently:

**Step 1: Check the Browser Console for Errors**

- Open the developer console: Press **F12** or right-click and choose **Inspect Element**.
- Look for errors: Identify any error messages related to the app's code.
- Analyze errors: Use the messages to pinpoint and fix problems within the code.

**Step 2: Verify API Usage**

- Confirm adherence to API documentation: Ensure API endpoints, methods, headers, and payloads are correct.
- Check authentication: Include necessary credentials or tokens.
- Validate parameters: Confirm required headers and request parameters are present.

**Step 3: Inspect App Preview**

- Review code for syntax errors: Look for obvious code mistakes.
- Check logic errors: Use breakpoints or `console.log` statements to trace execution.
- Isolate problematic sections: Narrow down where the code fails.

**Step 4: Assess Interface or Folder System Issues**

- Clear browser cache: Reload the Pro-code Editor after clearing the cache to rule out client-side issues.
- If the issue persists: Escalate to Level 2 support with detailed information and reproduction steps. Review the notes [below](#escalation_notes) before escalating.

**Step 5: Test with a Minimal Example**

- Create a minimal reproducible example: Simplify code to the smallest version that reproduces the issue.
- Use it to isolate the problem: Helps exclude external factors or complex code interactions.

**Step 6: Review Updates and Known Issues**

- Check for platform updates or release notes: Determine if recent changes might affect functionality.
- Visit the [Community Forums](https://community-forums.domo.com/main)
