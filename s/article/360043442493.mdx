## Steps to Create a Jira Ticket

1.    Project – Always be set to: Domo (DOMO)  
 2.    Issue Type – Set to: Bug  
 3.    Summary – Should start with “CUST:” then have a description of the bug ticket.  
 4.    Affects Version/s – Select the version of Domo the customer is on. (This can be found by going to: customer.domo.com/version)  
 5.    Components – Select the team the bug is for and the specific part of the product that is broken.  
 6.    Assignee – This will be the L2 who told you to create the bug.  
 7.    Squad – This will always be set to: Support  
 8.    Description – This is where you will use the bug template specific to the type of bug you are creating. (These templates can be found here: )  
 9.    Attachment – This is where you will add any screenshots/video of the issue or password protected zipped customer credential files (for Connector cases.)  
 10.    Reporter - This will be the L2 who told you to create the bug.  
 11.    Client Services – Click this tab to reach the second page of information to fill out.  
 12.    Customer Name – This is where you will put the customer's Domo domain. (For example, if their domain is target.domo.com you will put target in the field.)  
 13.    SalesForce Case Number – This is where you will put the SalesForce Case number that you are creating the bug for.  
 14.    Create – You will then click the create button to finish the bug ticket.

<Note>**Note:** There will be a pop-up window in the top right-hand side of your screen that contains the Jira number you just created. You must click this before it disappears, otherwise it is very difficult to find the ticket you just created.</Note>
